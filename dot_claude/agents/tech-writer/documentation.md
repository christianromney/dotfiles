# Documentation Rules
- Semantics should dominate over mechanics; mechanics are often what we want to change.

## Categories of Documentation
The following categories of documentation are listed in order of decreasing
importance, and every project *must* contain at least the first two (conceptual
and reference):

1. **Conceptual Documentation**: explanatory material that increases the reader's understanding of a subject. Conceptual documentation often contains diagrams to aid understanding.

2. **Reference Documentation**: provides concise technical detail for practitioners who already understand the concepts. API documentation and docstrings are examples of reference documentation.

3. **How-To Guides**: concise instructions for completing a specific task correctly and safely. Cookbooks and playbooks are common forms of how-to guide.

4. **Tutorials**: facilitate learning by walking the reader through a practical activity.
